01 · Problem Statement

The ₹8,000 Crore Problem

Indian pharmacies and hospital formularies lose an estimated ₹2,000–8,000 crore annually to drug expiry write-offs, overstock capital lock-in, and procurement inefficiency. The root cause is not negligence — it's that detection is manual and reaction is always late.

By the time a pharmacist notices an expired batch, the loss has already materialised. By the time a procurement manager spots overstock, capital has been locked for months. By the time an auditor flags a discount anomaly, revenue has already leaked.

Without an Agent

  • ❌ Expiry discovered after the batch turns bad
  • ❌ No rupee quantification of inventory risk
  • ❌ Overstock invisible until year-end audit
  • ❌ Discount abuse undetected across users
  • ❌ No audit trail linking signals to actions

With PharmaCost FinAgent

  • ✅ Breach detected before the financial hit lands
  • ✅ Every anomaly shown with exact ₹ math
  • ✅ Capital lock-in surfaced in real-time
  • ✅ Discount patterns flagged per user
  • ✅ Full SQL trigger audit trail auto-maintained
02 · Agentic Architecture

3 Agents. 1 Orchestrator. Full Loop.

📡 DETECT
→

Agent 1

Expiry Watchdog

  • Reads vw_ExpiredBatches, vw_ExpiringBatches
  • Classifies: expired / critical / warning
  • Quantifies: quantity × price = ₹ loss
  • Action: Auto-flag expired, Stage critical
  • Recovery: 0% expired, 60% critical, 85% warning
Agent 2

Spend Intelligence

  • Reads Batch, Drug, Bill, BillItem tables
  • Detects overstock, slow-movers, discount abuse
  • Quantifies: locked capital, revenue leakage
  • Action: Stage for procurement approval
  • Computes velocity: units sold / 30 days
Agent 3 + Orchestrator

Compliance Sentinel

  • Reads AuditLog, AgentActions
  • Detects audit gaps, regulatory breaches
  • Quantifies: 25% penalty on expired still in stock
  • Action: Escalate to admin immediately
  • Synthesises all 3 agents → Before/After model

Enterprise Approval Workflow

Every agent action is routed through a 3-tier approval system — matching real enterprise approval workflows exactly as the judges require:

⚡ Auto-Execute

Flag expired batch for disposal, write to audit log. No human approval needed — time-critical actions.

⏳ Stage for Approval

Procurement decisions, discount policy changes. Staged in AgentActions table — admin reviews and approves or rejects.

🚨 Escalate

Regulatory breaches, audit gaps, expired stock still on shelves. Hard escalation with immediate flag in audit log.

05 · Competitive Differentiation

Why This Beats Generic Dashboards

Most hackathon entries for Track 3 will be dashboards with pretty charts that show data. They will have a "financial summary" page with some numbers. The judges have explicitly said: "Pure mockups without data integration will score low."

PharmaCost Intelligence is different in 4 specific ways that judges can verify in the terminal, not just read about in a README:

1. Real signal → real action

Run python agents/orchestrator.py — see financial output in terminal immediately. No setup needed.

2. Math is verifiable

Every ₹ figure = quantity × price from actual DB rows. Judges can open the SQLite file and verify the arithmetic themselves.

3. Actions change the DB

Clicking "Execute" writes to AuditLog via SQL triggers. It's not a button that changes a colour — it actually records in the database.

4. Works on any dataset

Judges can upload their own CSV at /admin/import and the agent rescans immediately with their data — no code changes.

08 · Honest Assessment

How Strong Is This Entry?

This is a solid Track 3 submission — not a guaranteed winner, but competitive enough to place in the top third of entries if presented correctly.

The core strength is authenticity. The financial model is real math from a real database. The agents genuinely run queries and produce output. The approval workflow is enforced in code, not mocked in a slide. These are the things judges look for when they try to break demo entries.

The weakness is domain scope. A pharmacy is a small-scale version of the enterprise procurement problem the judges have in mind. Counter this in your pitch by emphasising that the architecture is enterprise-grade — the same agents can be pointed at any pharmaceutical supply chain, hospital formulary, or retail pharmacy chain with zero code changes.
